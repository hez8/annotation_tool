# 图像标注工具

用于图像的 Bounding Box 标注桌面工具。

## 功能

- **图片浏览**：打开本地文件夹，左右按钮或方向键快速切换图片
- **缩放标注**：支持放大/缩小/适应窗口/原始大小，滚轮缩放，便于标注细小物体
- **12 类别**：按钮式类别选择，点击后在图像上拖拽画框
- **框微调**：通过左/上/右/下四个参数独立调节标注框，支持步长配置
- **标注管理**：「下一个」存储当前标注继续标注，「确定」保存全部标注
- **归一化比例存储**：标注框以 0~1 归一化坐标存储，训练时图像缩放 GT 自动适配
- **自动加载**：打开图片时自动搜索同名 txt 并导入已有标注
- **自动保存**：切换图片时自动保存未存储的标注
- **多格式支持**：支持 jpg/png/bmp/tiff 等常见格式
- **TIFF 高位深适配**：自动将 16-bit / 32-bit / 浮点 TIFF 归一化到 8-bit 显示
- **3倍下采样模式**：针对多光谱 TIFF，可切换到 3倍 stride 下采样视图标注，坐标自动映射回原始尺寸
- **有效工作区域**：以 box 方式圈选有效工作区域，区域外图像视为无用区域不参与训练，以绿色虚线框 + 四角标线显示，支持微调模式精确调整，保存于 txt 文件中
- **双相机配准投影**：固定双相机场景下，只需标注相机A的图像，通过单应性矩阵自动将标注框投影到相机B的图像上，大幅减少标注工作量

## 环境依赖

- Python 3.7+
- Pillow（PIL）
- numpy
- tifffile（可选，用于 PIL 无法解析的科学/工业 TIFF 格式）

```powershell
conda activate py39
pip install Pillow numpy tifffile
```

## 启动方式

```powershell
conda activate py39
cd "标注目录"
python fod_annotation_tool.py
```

## 使用流程

```
打开文件夹 → 自动加载第一张图片
    ↓
[可选: 勾选"3倍下采样"切换视图]
    ↓
[可选: 点击「设置工作区域」拖拽绘制有效区域]
    ↓
[可选: 勾选工作区域「微调模式」精确调整区域边界]
    ↓
[点击类别按钮] 选择标注类别（如"金属"）
    ↓
[鼠标拖拽] 在图像上画框 → 红色虚线预览
    ↓
[微调] 通过左/上/右/下按钮或手动输入调整框位置
    ↓
[点击「下一个」] 标注存入结果列表 → 可继续选择其他类别标注
    ↓
... 重复以上步骤标注所有目标 ...
    ↓
[点击「确定 ✓」] 保存到同名 .txt 文件
    ↓
[◀▶] 切换图片，继续下一张
```

| 快捷键 | 功能 |
|--------|------|

| 鼠标滚轮 | 缩放图像 |
| `Ctrl` + `O` | 打开文件夹 |
| 长按 ◀/▶ | 按住微调按钮不放，以 40ms 间隔连续调整 |

## 3倍下采样模式

针对多光谱马赛克 TIFF 图像（如 2048×2048 原始图像包含 9 个波段），可勾选缩放栏的「3倍下采样」切换视图：

| 模式 | 显示尺寸 | 说明 |
|------|----------|------|
| 原始（默认） | 原始尺寸 | 直接标注原始 TIFF |
| 3倍下采样 | 原始 ÷ 3 | stride=3 采样，显示更清晰的光谱通道 |

两种模式下的标注坐标**自动映射**到原始图像尺寸，保存的 txt 始终基于原始图像。

## 标注文件格式

标注结果保存为与图片同名的 `.txt` 文件，内容为 JSON 对象，包含 `work_region`（可选）和 `annotations` 数组，bbox 以**归一化比例**（0~1）存储：

```json
{
  "work_region": {
    "x1": 0.05,
    "y1": 0.05,
    "x2": 0.95,
    "y2": 0.95
  },
  "annotations": [
    {
      "id": 0,
      "name": "金属",
      "bbox": {
        "x1": 0.0625,
        "y1": 0.212963,
        "x2": 0.164062,
        "y2": 0.425926
      }
    },
    {
      "id": 3,
      "name": "木头",
      "bbox": {
        "x1": 0.260417,
        "y1": 0.296296,
        "x2": 0.354167,
        "y2": 0.472222
      }
    }
  ]
}
```

| 字段 | 说明 |
|------|------|
| `work_region` | （可选）有效工作区域，归一化坐标。不设置时该字段无。区域外图像不参与训练 |
| `annotations` | 标注列表 |
| `id` | 类别编号（0-11） |
| `name` | 类别中文名称 |
| `bbox.x1` / `y1` / `x2` / `y2` | 归一化坐标（0~1），相对原图宽/高 |

> **优势**：训练时对图像做任意缩放，GT 始终正确。兼容旧版像素坐标格式（加载时自动检测转换）。旧版纯数组格式 txt 文件仍可正常加载。

## TIFF 高位深适配

工具会自动检测并处理以下 TIFF 格式：

| 模式 | 位深 | 处理方式 |
|------|------|----------|
| `I;16` | 16-bit 灰度 | getextrema() → 拉伸到 8-bit |
| `I` / `F` | 32-bit 灰度/浮点 | getextrema() → 拉伸到 8-bit |
| 科学/工业 TIFF | 非标准编码 | tifffile 兜底读取 → numpy 归一化 |

## 修改标注

- **双击标注结果列表**中的某一行，可将该标注加载到画布上进行修改（框会重新显示为红色预览状态），修改后点击「下一个」重新存储
- 选中列表中的标注行，点击「删除选中」可删除该条标注

点击右侧「选择...」按钮可指定标注文件保存到其他目录。默认保存到图片所在目录。

## 有效工作区域

用于圈选图像中真正有效的区域。区域外的图像部分在训练时将被忽略，避免无用背景干扰模型学习。

### 设置工作区域

1. 点击「设置工作区域」按钮，进入绘制模式
2. 在图像上拖拽绘制绿色虚线框
3. 松手后自动进入**微调模式**，可使用左侧微调面板精确调整边界
4. 点击「清除」按钮可删除工作区域

### 视觉效果

- **绿色虚线边框** + **四角 L 形标线**，与标注框（蓝色实线）和预览框（红色虚线）明确区分
- 保存后 txt 文件包含 `work_region` 字段（归一化坐标）

### 微调模式

勾选「微调模式」复选框后，左侧的标注框微调面板切换为控制工作区域：
- 使用 ◀/▶ 按钮或手动输入调整 x1/y1/x2/y2
- 按钮支持**长按连续调整**（按住不放以 40ms 间隔连续变化）
- 开始画新标注框时自动退出微调模式
- 清除工作区域时也会自动退出

### 工作区域投影

使用双相机配准投影功能时，源图的工作区域也会通过单应性矩阵变换到目标图：
- 工作区域角点与标注框一样经过变换 → 钳位到目标图范围
- 投影成功后状态栏提示 "已投影 N 条标注 + 工作区域"

## 双相机配准投影

适用于**固定双相机**拍摄场景。两个相机同时拍摄同一物体，分别存入两个文件夹。只需标注相机A的图像，标注框通过单应性矩阵自动投影到相机B的图像上，大幅减少标注工作量。

### 前提条件

- 两个相机相对位置**固定不变**（如固定在同一个支架上）
- 两个相机的图片分别存放在**不同文件夹**中，但**文件名相同**（如 `camA/img001.jpg` 和 `camB/img001.jpg`）
- 需要至少一对对应图片用于标定，建议选特征点明显、覆盖四角的图片

### 整体流程概览

```
┌─────────────────────────────────────────────────────────────┐
│  一次性标定（选一对代表性图片）                                │
│                                                             │
│  打开文件夹A → 选一张图 → 采源图特征点(4+)                    │
│     ↓                                                       │
│  菜单「文件→打开文件夹」切换到文件夹B → 选同一场景的对应图      │
│     ↓                                                       │
│  采目标图特征点(按相同顺序) → 计算并保存 homography.json       │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  日常标注（每对图片重复）                                      │
│                                                             │
│  打开文件夹A → 逐张标注 → 保存 txt                            │
│     ↓                                                       │
│  菜单「文件→打开文件夹」切换到文件夹B → 打开对应图片            │
│     ↓                                                       │
│  加载矩阵(首次) → 从源图投影标注 → 检查/微调 → 保存            │
└─────────────────────────────────────────────────────────────┘
```

### 详细操作步骤

#### 第一阶段：标定（仅需做一次，约 2 分钟）

> 标定的目的是计算出相机A图像到相机B图像的变换矩阵。因为相机是固定的，算好之后永久复用。

**步骤 1：打开相机A的文件夹并选一张图**

- 菜单「文件 → 打开文件夹」→ 选择相机A的图片文件夹
- 左右箭头切换到一张**特征点明显**的图片（如场景中有明显角点、标记物）
- 特征点选择建议：物体的角、地面标记、明显的边缘交点等

**步骤 2：在相机A的图上采集特征点**

- 将右侧面板向下滚动，找到「配准变换 (双相机投影)」区域
- 点击「采源图点」按钮（按钮呈按下状态）
- 在图像上依次点击 **4 个以上** 特征点，顺序任意但需记住
  - 每次点击出现**黄色圆形 + 编号标记**（①、②、③、④...）
  - 点越多越好（建议 5~8 个），尽量**分散覆盖图像四角**
- 点完后按 **ESC** 或再次点击「采源图点」退出
- 右侧面板会显示 "源:4 目标:0"（表示已采集 4 个源图点）

**步骤 3：切换到相机B的文件夹并选对应图片**

- 菜单「文件 → 打开文件夹」→ 选择相机B的图片文件夹
- 左右箭头切换到**同一场景**的对应图片（建议与步骤1文件名相同）
- > **注意**：切换文件夹后配准点数据会自动保留，但视觉标记会清除。右侧面板仍显示 "源:4 目标:0"

**步骤 4：在相机B的图上采集对应特征点**

- 点击「采目标图点」按钮
- 此时画布**右下角会出现源图参考缩略图**，黄色编号标记显示之前采集的源图特征点位置
- **按照与源图相同的顺序**，在目标图上点击对应的物理特征点
  - 例如：源图第①个点选的是"桌面左上角"，目标图第①个点也选"桌面左上角"
  - 缩略图上的编号帮助你回忆点击顺序和位置
- 点完后按 ESC 或再次点击按钮退出

**步骤 5：计算并保存矩阵**

- 点击「计算并保存矩阵」按钮
- 在弹出的保存对话框中选择保存位置（建议保存在项目根目录，如 `D:\数据集\homography.json`）
- 程序弹出提示框显示计算出的 3×3 矩阵
- 右侧面板显示 "矩阵: 0.9512 0.0234 ..." 表示已就绪

> **标定完成！** `homography.json` 可永久复用，下次启动软件直接「加载矩阵」即可。

---

#### 第二阶段：日常标注（每对图片重复）

**步骤 1：标注相机A的图片**

- 菜单「文件 → 打开文件夹」→ 选择相机A的图片文件夹
- 逐张进行常规标注（选类别 → 画框 → 微调 → 下一个 → 确定保存）
- 确保每张图都保存了 txt 标注文件

**步骤 2：加载变换矩阵**

- 菜单「文件 → 打开文件夹」→ 切换到相机B的图片文件夹
- 打开第一张需要投影的图片
- 在「配准变换」面板中点击「加载矩阵」
- 选择之前保存的 `homography.json` 文件
- 右侧面板显示 "矩阵: 0.9512 0.0234 ..." 确认加载成功

> 矩阵只需加载一次，切换图片时矩阵保留在内存中。

**步骤 3：投影标注（两种方式）**

*方式A — 批量自动投影（推荐）*
- 点击「源标注文件夹」→ 选择相机A的标注 txt 所在文件夹
- 勾选「自动投影」复选框
- 切换目标图片时，自动按**文件名匹配**找到对应源 txt 并投影
- 状态行显示当前源文件夹路径和匹配的 txt 文件名
- 当前图片已有标注时弹窗确认是否覆盖
- 超出最后一个 txt 时弹窗提示序列结束

*方式B — 手动单张投影*
- 点击「从源图投影标注」→ 手动选择相机A的标注 txt 文件
- 程序自动将源标注框和**工作区域**变换并导入
- 弹出提示框显示 "成功投影 N 条标注 + 工作区域"

**步骤 4：检查并保存**

- 检查投影后的标注框和工作区域是否准确
- 如需微调：双击标注结果列表中的条目 → 画布上出现红色预览框 → 用微调面板调整 → 点击「下一个」重新存储
- 点击「确定 ✓」保存到目标图的 txt 文件
- 批量模式下切换到下一张目标图自动投影

### 配准点操作技巧

- **撤销上一点**：点击「撤销上一点」按钮，只删除最近采集的一个点（不会全部清除）。误点击时无需从头开始。
- **方向键微调**：在配准采点模式下，`← → ↑ ↓` 键以 1 像素步长微调最近采集的配准点坐标。
- **微调面板调整**：配准模式下，左侧微调面板自动显示最近配准点坐标，可使用 ◀▶ 按钮（支持长按）或手动输入精确调整。
- **清除全部点**：点击「清除全部点」重置所有已采集的配准点。

### 源图参考缩略图

在目标图上采集配准点时，画布**右下角**会自动显示源图的缩略图：

- 深色背景 + 黄色边框 + "源图参考 (按序点击对应点)" 标题
- 黄色编号标记显示之前采集的源图特征点位置
- 帮助你在目标图上按相同顺序点击对应物理点
- 这个缩略图仅在配准模式（采目标图点）时显示

### 跨文件夹操作要点

| 操作 | 说明 |
|------|------|
| 切换文件夹 | 菜单「文件 → 打开文件夹」，不会丢失配准点数据和矩阵 |
| 配准点保留 | 切换文件夹/图片时，已采集的源图点和目标图点**保留在内存中** |
| 矩阵保留 | 加载或计算出的矩阵在软件运行期间始终有效 |
| 文件名对应 | 建议两个文件夹中的图片使用**相同文件名**，方便配对 |

### 工作原理

- **DLT 算法**：使用 SVD 分解求解 3×3 单应性矩阵，纯 numpy 实现，无需 OpenCV
- **Bbox 变换**：将标注框四个角点通过矩阵变换，再取轴对齐包围盒（AABB）作为新标注框
- **退化框过滤**：变换后尺寸过小（<3像素）的框自动丢弃
- **坐标自动处理**：投影时自动从标注文件读取原图分辨率，归一化坐标自动转像素坐标再变换

### homography.json 格式

```json
{
  "matrix": [
    [0.9512, 0.0234, 10.5678],
    [-0.0123, 0.9715, -5.2341],
    [0.0001, -0.00005, 1.0]
  ],
  "dst_resolution": [2048, 1536]
}
```

### 加载已有矩阵

如果之前已经标定并保存过矩阵文件：
1. 打开任意目标图片
2. 点击「加载矩阵」→ 选择 `homography.json`
3. 直接使用，无需重复标定

## 标注框微调增强

### 长按连续调整

微调面板的 ◀/▶ 按钮支持**长按**操作：
- **单击**：调整 1 个步长（与传统行为一致）
- **按住不放**：首次延迟 200ms 后，以 40ms 间隔连续调整，松开或鼠标移出即停
- 适用于需要大幅度调整标注框/工作区域/配准点的场景

### 尺寸信息显示

微调面板底部实时显示当前编辑对象的尺寸：
- **标注框** / **工作区域**：显示 宽 (x2-x1)、高 (y2-y1)、面积 (宽×高)，蓝色字体
- **配准点**：显示 "配准点坐标"
- **无编辑对象**：显示 "宽: — 高: — 面积: —"
- 拖拽画框、点击按钮、手动输入时同步更新

### 配准点微调

在采集配准点时，提供三种微调方式互补使用：
| 方式 | 操作 | 步长 |
|------|------|------|
| 方向键 | `← → ↑ ↓` | 1 像素 |
| 微调面板 ◀▶ | 点击/长按 | 跟随微调步长设置 |
| 手动输入 | 输入框回车 | 精确值 |

## 保存目录自动刷新

切换保存目录后，工具会自动检测新目录下是否有同名 txt 文件：
- 有同名 txt → 自动加载标注并重绘画布
- 无同名 txt → 保留当前标注，状态栏提示无已有标注
- 菜单「文件 → 选择保存目录」和右侧「选择...」按钮均可触发
